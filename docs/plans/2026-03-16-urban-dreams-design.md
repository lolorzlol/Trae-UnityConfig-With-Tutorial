# 《都市梦华录》- 游戏设计文档（优化版）

## 📋 游戏概述

**一款休闲风格的现代都市经营模拟游戏**，玩家将从一个小型创业者起步，通过商业投资、房地产开发、城市建设，逐步打造属于自己的都市帝国。游戏节奏轻松，画风温馨，适合在空闲时间放松游玩。

**平台**：PC (Steam)  
**类型**：策略/经营  
**风格**：休闲轻松  
**模式**：纯单机

---

## 🎯 一、MDA 框架设计

### Mechanics（机制层）

| 机制类别 | 具体规则 | 设计目的 |
|----------|----------|----------|
| **资源管理** | 资金、时间、人力、原材料 | 创造决策深度 |
| **建筑系统** | 购买/建造/升级/拆除 | 提供策略选择 |
| **经济系统** | 收入、支出、税收、投资 | 建立经济循环 |
| **员工系统** | 雇佣、解雇、培训、薪资 | 增加管理维度 |
| **科技系统** | 解锁新建筑和新功能 | 提供成长感 |

### Dynamics（动态层 - 涌现行为）

```
基础机制交互 → 涌现的游戏行为
建筑布局 + 建筑类型 → 最优收益布局策略
居民满意度 + 税收率 → 玩家决策权衡
员工效率 + 薪资成本 → 人力资源优化
股票市场 + 并购系统 → 风险收益博弈
```

### Aesthetics（体验层 - 情感目标）

| 美学体验 | 设计实现 |
|----------|----------|
| **成就感** | 阶段目标达成动画、金币特效、成就解锁 |
| **掌控感** | 自由建设、随意规划、无惩罚的试错 |
| **归属感** | 居民互动、城市繁荣的视觉反馈 |
| **惊喜感** | 随机事件、节日活动、隐藏彩蛋 |

## 🧠 二、玩家心理设计

### 2.1 Bartle 玩家类型适配

| 玩家类型 | 核心动机 | 本游戏对应设计 |
|----------|----------|----------------|
| **Achievers（成就者）** | 完成目标、收集、解锁 | 100+成就系统、里程碑奖励、排行榜 |
| **Explorers（探索者）** | 发现秘密、挖掘隐藏内容 | 隐藏区域、随机事件、蓝图收集 |
| **Socializers（社交者）** | 与其他玩家互动 | 游客参观模式、共享蓝图（未来） |
| **Killers（杀手）** | 竞争、超越他人 | 成就排行榜、效率竞争 |

### 2.2 自我决定理论（SDT）

```
┌─────────────────────────────────────────────────────────┐
│                    SDT 三大需求                          │
├─────────────────────────────────────────────────────────┤
│  AUTONOMY（自主性）                                      │
│  → 自由选择发展路径：商业为主 / 地产为主 / 混合发展     │
│  → 建筑布局无限制                                      │
│  → 无强制主线任务                                      │
├─────────────────────────────────────────────────────────┤
│  COMPETENCE（胜任感）                                    │
│  → 清晰的目标体系：4个阶段目标                         │
│  → 即时反馈：收入数字跳动、建设进度                    │
│  → 可达成的挑战：难度递增但平滑                        │
├─────────────────────────────────────────────────────────┤
│  RELATEDNESS（归属感）                                   │
│  → 居民系统：满意度、需求反馈                          │
│  → 城市发展：人口增长的可视化                          │
│  → 员工互动：雇佣关系                                  │
└─────────────────────────────────────────────────────────┘
```

### 2.3 心流通道设计

```
        焦虑
         ↑ 
  挑战  │     ████████
  较高  │   ████████████  ← 最佳区间（略高于当前能力）
        │ ███████████████
        │██████████████████████
  挑战  │█████████████████████████
  较低  │████████████████████████████
        └────────────────────────────→ 技能/经验
        
        玩家初期 → 玩家中期 → 玩家后期
        (引导)    (探索)    (精通)
```

**设计策略**：
- 初期：强制引导教程，降低认知负担
- 中期：开放更多系统，让玩家自主探索
- 后期：提供深度优化空间（追求效率最大化）

## 🎁 三、奖励系统设计

### 3.1 奖励类型矩阵

| 奖励类型 | 例子 | 时机 | 心理效应 |
|----------|------|------|----------|
| **即时反馈** | 收入金币动画 | 每秒/每次交易 | 强化行为 |
| **里程碑** | 达成阶段目标 | 达到条件时 | 成就感 |
| **随机惊喜** | 随机事件奖励 | 随机触发 | 惊喜感 |
| **收集奖励** | 成就解锁 | 完成成就 | 完整性动机 |
| **社交奖励** | 游客点赞 | 被访问时 | 认可需求 |

### 3.2 奖励节奏设计

```
固定比率（Fixed Ratio）：
  → 每卖出一个商品获得金币
  → 保持频繁的积极反馈

变量间隔（Variable Interval）：
  → 随机事件：每 30-120 秒可能触发
  → 保持玩家持续关注

里程碑奖励（Milestone）：
  → 阶段目标达成：获得大量金币+解锁内容
  → 长线目标激励

成长曲线：
  奖励频率：初期高 → 中期稳定 → 后期需要玩家主动追求
  奖励价值：初期高倍率 → 后期边际递减
```

### 3.3 边际递减机制

| 建筑数量 | 单个收益倍率 | 设计原因 |
|----------|--------------|----------|
| 1-3 | 100% | 鼓励初期发展 |
| 4-6 | 85% | 轻微递减 |
| 7-10 | 70% | 促使多元化 |
| 11+ | 60% | 鼓励探索其他系统 |

## ⚖️ 四、平衡原则

### 4.1 经济平衡表

| 资源 | 产出/周期 | 消耗/周期 | 平衡策略 |
|------|-----------|-----------|----------|
| 资金 | 商业收入 | 建筑维护/薪资 | 基础收入 > 基础支出 |
| 原材料 | 采购/产出 | 建筑建造 | 随等级解锁采购渠道 |
| 人力 | 雇佣 | 薪资/培训 | 效率 vs 成本权衡 |
| 时间 | 游戏加速 | 无 | 可选付费加速 |

### 4.2 风险-收益矩阵

| 投资类型 | 风险等级 | 预期收益 | 流动性 |
|----------|----------|----------|--------|
| 银行存款 | 0% | 2%/天 | 高 |
| 稳健投资 | 10% | 5%/天 | 中 |
| 激进投资 | 30% | 12%/天 | 低 |
| 并购 | 50% | 20%+ | 低 |

### 4.3 难度曲线

```

  │    ╭─────── 后期：深度优化
  │   ╱
  │  ╱  中期：多系统开放
  │ ╱
  │╱   初期：简单引导
 ├─────────────────────→ 时间/进度
```

## 🔄 五、核心循环设计

### 5.1 主循环

```
         ┌──────────────────────────────────────────┐
         │              核心循环                      │
         │  ┌─────┐    ┌─────┐    ┌─────┐           │
         │  │输入 │ → │处理 │ → │输出 │           │
         │  └─────┘    └─────┘    └─────┘           │
         │     ↑                      ↓             │
         │     │    ┌─────────────┐   │             │
         │     └────┤   奖励反馈  ├───┘             │
         │          └─────────────┘                 │
         └──────────────────────────────────────────┘
```

### 5.2 玩家体验循环

```
第1层：即时满足
  建造建筑 → 立即显示效果 → 获得收入

第2层：短期目标
  赚够钱 → 解锁新建筑 → 扩大规模

第3层：中期成就
  达成阶段目标 → 获得里程碑奖励 → 解锁新功能

第4层：长期追求
  收集所有成就 → 打造完美城市 → 探索隐藏内容
```

## 🏙️ 核心玩法

### 1. 商业经营
- 初期经营小店铺（咖啡店、便利店、书店等）
- 随着资金积累，可扩展为连锁店
- 员工管理：雇佣员工，设置薪资，提升效率

### 2. 房地产开发
- 购买空地，建设住宅、写字楼、商业综合体
- 根据区块特性选择合适的建筑类型
- 升级改造已有建筑，提升价值

### 3. 城市发展
- 吸引居民入住，提升城市人口
- 建设基础设施（公园、学校、医院、地铁站）
- 居民满意度影响税收和商业收入

### 4. 投资并购
- 股票市场：投资其他企业，获取分红
- 并购小公司，扩大商业版图
- 竞标政府大型项目

## 🎨 美术风格

- **2.5D 等距视角**（类似《模拟城市3000》）
- **清新卡通风格**：柔和的色彩，温暖的灯光
- **白天/夜晚循环**：城市呈现不同的面貌
- **UI设计**：简洁直观，信息清晰，符合 Fitts 定律（大按钮、易点击）
